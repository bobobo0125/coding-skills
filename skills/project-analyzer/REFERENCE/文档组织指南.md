# 项目分析器 - 文档组织指南

## 文件组织结构

每个项目分析都会创建一个结构化的文档目录，方便管理和查阅。

```

    ├── [项目名]-progress-tracking.md  # 分析进度跟踪
    ├── analysis-todo.md               # 分析TODO列表
```

## 文件命名规范

### 主文档
- **综合报告**：`[项目名]-analysis.md`
- **进度跟踪**：`[项目名]-progress-tracking.md`

### 主题文档
格式：`XX-[主题名].md`
- `XX`：两位数序号（01-20）
- `[主题名]`：简化主题名

**示例**：
```
01-project-basic-info.md
13-source-code-deep-dive.md
```

### 图表文件
格式：`[图表类型]-[描述].mmd`
- `architecture-overview.mmd`
- `sequence-user-flow.mmd`
- `flowchart-data-processing.mmd`

## 文档管理最佳实践

### 自动创建
1. 每个主题完成后立即创建相应主题文档
2. 同时更新进度跟踪文档
3. 更新主综合报告中的相关部分

### 文档链接
- 使用相对链接连接相关文档
- 主题文档之间相互引用
- 主报告链接到所有主题文档

### 版本控制
- 每个文档底部包含生成时间戳
- 重大更新记录更新历史
- 保持文档版本一致性

### 质量控制
- 使用统一的模板和格式
- 确保所有链接有效
- 验证图表渲染正确

这种渐进式文档创建策略确保：
1. **实时保存**：每个主题完成后立即保存，防止数据丢失
2. **灵活查看**：可以单独查看任何主题的详细分析
3. **进度透明**：清晰显示分析进度和状态
4. **易于维护**：结构化组织便于后续更新和扩展
