## 1. 命名规范

### 1.1 类命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 类/接口 | UpperCamelCase | `UserService`, `MarketController` |
| 抽象类 | Abstract/Base开头 | `AbstractDao` |
| 枚举 | 以Enum结尾 | `MarketStatusEnum` |
| 异常类 | Exception结尾 | `BusinessException` |
| DO/BO/DTO/VO | 保持原样式 | `UserDO`, `OrderDTO` |

### 1.2 方法命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 查询方法 | find/select/get + 业务名 | `findById`, `findByName` |
| 插入方法 | save/insert + 业务名 | `saveUser`, `insertOrder` |
| 更新方法 | update + 业务名 | `updateUser` |
| 删除方法 | remove/delete + 业务名 | `removeById`, `deleteUser` |
| 条件判断 | is/has/can + 特性 | `isActive`, `hasPermission` |

### 1.3 变量命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 普通变量 | lowerCamelCase | `userName`, `marketList` |
| 常量 | UPPER_SNAKE_CASE | `MAX_COUNT`, `DEFAULT_STATUS` |
| 成员变量 | lowerCamelCase（加前缀m_也可） | `mUserName` |
| 参数 | lowerCamelCase | `userRequest`, `marketId` |

## 2. 代码格式

### 2.1 缩进与空格

```
✅ 缩进：4个空格（禁止Tab）
✅ 左大括号前不换行
✅ 右大括号后换行
✅ 运算符前后加空格
✅ 逗号后加空格
✅ 注释//后加空格
```

```java
// ✅ Good
if (condition) {
    doSomething();
}

int a = 1 + 2;
for (int i = 0; i < 10; i++) {
    // comment
}
```

### 2.2 类结构顺序

```java
public class UserService {

    // 1. 常量
    private static final Logger log = LoggerFactory.getLogger(UserService.class);
    private static final int MAX_SIZE = 100;

    // 2. 成员变量
    private UserRepository userRepository;

    // 3. 构造方法
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // 4. 公开方法
    public void doSomething() {}

    // 5. 受保护方法
    protected void helper() {}

    // 6. 私有方法
    private void internal() {}
}
```

## 3. 注释规范

### 3.1 JavaDoc要求

```java
/**
 * 用户服务类
 * 负责用户的CRUD操作、权限验证等
 *
 * @author developer
 * @since 1.0.0
 */
public class UserService {}

/**
 * 根据ID查询用户
 *
 * @param id 用户ID
 * @return 用户信息
 * @throws UserNotFoundException 用户不存在
 */
public User findById(Long id) {}
```

### 3.2 注释位置

```java
// ✅ Good
if (condition) {
    // 执行初始化操作
    initialize();
    // 执行核心逻辑
    process();
}

// ❌ Bad
if (condition) { // 初始化操作
    initialize();
}
```

## 4. 控制语句

### 4.1 必须使用大括号

```java
// ✅ Good
if (condition) {
    return result;
}

// ❌ Bad
if (condition)
    return result;
```

### 4.2 避免复杂条件

```java
// ✅ Good
boolean isValid = validateInput(param);
if (isValid) {
    process();
}

// ❌ Bad
if (validateInput(param) && process()) {}
```

### 4.3 switch语句

```java
// ✅ Good - 必须有default
switch (status) {
    case ENABLED:
        enable();
        break;
    case DISABLED:
        disable();
        break;
    default:
        log.warn("未知状态: {}", status);
}
```

## 5. 异常处理

### 5.1 不能生吞异常

```java
// ✅ Good
try {
    doSomething();
} catch (Exception e) {
    log.error("操作失败", e);
    throw new BusinessException("操作失败");
}

// ❌ Bad - 生吞异常
try {
    doSomething();
} catch (Exception e) {
    // 什么也不做
}
```

### 5.2 异常类型选择

| 场景 | 异常类型 |
|------|---------|
| 业务异常 | 非受检异常 (RuntimeException) |
| 参数校验 | IllegalArgumentException |
| 空值检查 | NullPointerException |
| 不支持操作 | UnsupportedOperationException |

### 5.3 异常封装

```java
// 业务异常定义
public class BusinessException extends RuntimeException {

    private final String errorCode;

    public BusinessException(String errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }
}
```

## 6. 日志规范

### 6.1 占位符用法

```java
// ✅ Good
log.info("查询用户 id={}", userId);
log.error("操作失败 id={}, error={}", id, e.getMessage(), e);

// ❌ Bad - 字符串拼接
log.info("查询用户 id=" + userId);
```

### 6.2 日志级别

| 级别 | 使用场景 |
|------|---------|
| ERROR | 影响功能的异常，需要立即处理 |
| WARN | 不影响功能但需要关注的问题 |
| INFO | 关键业务流程节点 |
| DEBUG | 调试信息，生产环境关闭 |

### 6.3 禁止输出敏感信息

```java
// ❌ Bad - 禁止记录敏感信息
log.info("用户密码: {}", password);
log.info("信用卡号: {}", cardNumber);
```

## 7. 并发处理

### 7.1 线程池命名

```java
// ✅ Good
ThreadFactory threadFactory = new ThreadFactoryBuilder()
    .setNamePrefix("market-pool-")
    .setDaemon(true)
    .build();
ExecutorService executor = new ThreadPoolExecutor(
    corePoolSize, maxPoolSize, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1000), threadFactory
);

// ❌ Bad
ExecutorService executor = Executors.newFixedThreadPool(10);
```

### 7.2 线程安全

```java
// ✅ Good - 使用ThreadLocal
public class UserContext {
    private static final ThreadLocal<String> USER = new ThreadLocal<>();

    public static void setUser(String user) {
        USER.set(user);
    }

    public static String getUser() {
        return USER.get();
    }

    public static void clear() {
        USER.remove();
    }
}
```

### 7.3 同步控制

```java
// ✅ Good - 使用同步块
private final Object lock = new Object();

public void method() {
    synchronized (lock) {
        // 临界区代码
    }
}
```

## 8. 数据库操作

### 8.1 SQL规范

```sql
-- ✅ 使用具体列名
SELECT id, name, status FROM t_user WHERE status = 'ENABLED';

-- ❌ 避免 SELECT *
SELECT * FROM t_user;
```

### 8.2 索引规范

```java
// ✅ Good - 索引命名
@Table(name = "t_user", indexes = {
    @Index(name = "idx_name", columnList = "name"),
    @Index(name = "idx_status", columnList = "status")
})
```

### 8.3 事务规范

```java
// ✅ Good - 明确事务边界
@Transactional(rollbackFor = Exception.class)
public void method() {
    // 业务逻辑
}
```

## 9. 集合处理

### 9.1 初始化

```java
// ✅ Good
List<String> list = new ArrayList<>();
Map<String, Object> map = new HashMap<>();

// 使用集合工厂方法（Java 9+）
List<String> list = List.of("a", "b", "c");
Set<String> set = Set.of("a", "b");
Map<String, Object> map = Map.of("key", "value");
```

### 9.2 遍历

```java
// ✅ Good - 使用for-each
for (String item : list) {
    process(item);
}

// ✅ Good - 使用Stream
list.stream()
    .filter(Objects::nonNull)
    .map(String::toUpperCase)
    .collect(Collectors.toList());
```

### 9.3 空集合返回

```java
// ✅ Good - 返回空集合而不是null
public List<User> findAll() {
    if (users == null) {
        return Collections.emptyList();
    }
    return users;
}
```

## 10. 日期处理

### 10.1 日期类型选择

| 场景 | 推荐类型 |
|------|---------|
| 存储日期时间 | LocalDateTime |
| 仅日期 | LocalDate |
| 仅时间 | LocalTime |
| 时间戳 | Instant |

### 10.2 日期格式化

```java
// ✅ Good
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String dateStr = LocalDateTime.now().format(formatter);

// ❌ Bad - 使用Date类
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
```
